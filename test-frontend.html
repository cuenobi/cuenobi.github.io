FLOWFUSION TECHNOLOGY – Front-End Code Challenge

This page outlines the front-end coding assignment for candidates applying for the role of Front-End Engineer or Full-Stack Engineer at FlowFusion Technology.

Please read all requirements thoroughly and build the assignment as if it were part of a real production system.

Goal: Build a responsive, production-ready web interface using React or Next.js that includes login, registration, and logout functionality.

1. Problem Overview

You are required to build a small front-end application with the following features:

1.1 Functional Requirements

  • A Login page
  • A Register page
  • A Logout function
  • A protected user area/page that is accessible only when logged in
  • Use of client-side storage for authentication state (localStorage, sessionStorage, or cookies)

You may connect the UI to a mock API, a local JSON server, a public mock API service, or your own temporary backend.

1.2 UI Expectations

  • Your UI should be clean, responsive, and easy to use.
  • You may use any component library (e.g. MUI, Ant Design, TailwindCSS, Chakra UI), or write custom components.
  • You should show clear validation and error messages when fields are invalid.

2. Technical Requirements

The project must use one of the following:

  • React (Vite or CRA)
  • Next.js (recommended)

State management is flexible (useState, Context API, Redux, Zustand, Jotai, etc.).

3. Bonus Features

3.1 Bonus: E2E Tests with Cypress (Optional)

Implement end-to-end tests using Cypress. Suggested test cases:

  • Login flow
  • Registration flow
  • Authenticated route redirection behavior
  • Logout flow

This is optional, but completing it will be considered a plus.

4. Project Setup & Deliverables

Your repository should include:

  • A functioning React or Next.js project
  • A clear README.md containing:
    • Project explanation
    • Tech stack used
    • Setup and run instructions
    • (Optional) Cypress test instructions
  • Well-structured React/Next.js code
  • Reusable and readable components

5. Non-Functional Requirements

We will evaluate your solution based on:

  • Code readability – clean structure, logical grouping, and consistent naming.
  • Simplicity – avoid unnecessary complexity.
  • User experience – intuitive navigation and responsive layout.
  • Performance awareness – fast rendering and efficient state management.
  • Security awareness – proper handling of authentication data.

6. Bonus: E2E Tests

If you choose to provide Cypress tests, they should cover at minimum:

  • Login flow
  • Registration flow
  • Protected route behavior
  • Logout flow

7. Submission Guidelines

  • Host your solution in a Git repository (GitHub, GitLab, or similar).
  • Ensure that your code is accessible to our evaluation team.
  • Include all setup instructions in the project README.
